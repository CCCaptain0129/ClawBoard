# TASK-015 飞书 API 问题分析

## 问题描述
Agent 卡片依然显示 ID 号码，没有显示群组名称。

---

## 测试结果
- **状态**: ❌ 失败
- **测试时间**: 2026-03-11 15:29 GMT+8

---

## 发现的问题

### 问题 1：数据源缺失
**OpenClaw Gateway 返回的 session 数据中没有 `to` 字段**

- 后端代码期望从 `session.to` 获取 chat_id（格式：`chat:oc_xxx`）
- 但 Gateway 实际返回的 session 数据中不包含此字段

**影响**: 无法获取群组 chat_id，导致无法调用飞书 API

### 问题 2：API 认证失败
**飞书 API 返回错误**: `{"code":10014,"msg":"app secret invalid"}`

- 说明当前的 `app_secret` 配置无效
- 或权限未正确开通

**影响**: 无法获取飞书访问令牌，无法调用群组信息 API

---

## 根本原因分析

TASK-015 功能无法正常工作的根本原因：

1. **数据源问题**
   - OpenClaw Gateway 的 sessions.list 返回的数据结构与后端代码期望的不一致
   - 缺少 `session.to` 字段

2. **认证问题**
   - 飞书应用的 `app_secret` 无效
   - 或权限未配置在正确的应用上

---

## 解决方案

### 方案 1：修复数据源问题
1. 检查 OpenClaw Gateway 返回的 session 数据结构
2. 确认群组信息存储在哪个字段（可能是 `session.key` 或 `session.id`）
3. 更新 `agentMonitor.ts` 中的代码，从正确的字段提取群组 ID

### 方案 2：修复认证问题
1. 确认飞书应用的 `app_secret` 是否正确
2. 确认 im:chat 权限是否配置在正确的应用上
3. 更新环境配置文件中的凭证

### 方案 3：使用降级方案
如果无法修复飞书 API，可以使用以下降级方案：
- 从 session 的其他字段提取更友好的显示名称
- 添加手动配置的群组名称映射
- 显示更友好的占位符
