# PMW-022: Subagent 完成自动补齐方案

## 🎯 目标

解决"Subagent 已结束但任务仍显示 in-progress"的问题，通过监控服务自动检测和补齐已完成的 Subagent 任务状态。

## ✅ 实现内容

### 1. SubagentMonitorService

**位置**: `src/backend/src/services/subagentMonitor.ts`

**核心功能**:
- ✅ 每 30 秒检查 `SUBAGENTS任务分发记录.md` 中仍为"进行中"的 subagentId
- ✅ 结合 OpenClaw sessions store（`~/.openclaw/agents/main/sessions/sessions.json`）判断是否已结束
- ✅ 若结束则调用 `SubagentManager.markSubagentComplete` 并更新任务状态
- ✅ 实现幂等性（使用内存缓存避免重复处理）
- ✅ 避免误判（>=2 分钟无更新才判定为已完成）

### 2. 后端集成

**位置**: `src/backend/src/index.ts`

- ✅ 在后端服务启动时自动启动 `SubagentMonitorService`
- ✅ 在服务停止时正确清理资源

## 🔧 核心逻辑

### 判定条件

Subagent 被判定为"可能已完成"需要满足以下任一条件：

1. **Session 不存在**：在 `sessions.json` 中找不到对应的 session
2. **Session 超时**：Session 存在但最后更新时间 >= 2 分钟前

### 工作流程

```
每 30 秒
  ↓
读取 SUBAGENTS任务分发记录.md
  ↓
提取所有状态为 🔄 进行中 的 subagentId
  ↓
对每个 subagentId:
  1. 读取 sessions.json
  2. 查找对应 session
  3. 检查最后更新时间
  4. 判断是否 >= 2 分钟无更新
  ↓
如果判定为已完成:
  1. 跳过已处理的（幂等性）
  2. 调用 markSubagentComplete()
  3. 更新记录文件
  4. 更新任务状态
```

### 配置参数

在 `src/backend/src/index.ts` 中可以配置：

```typescript
const subagentMonitorService = new SubagentMonitorService({
  intervalMs: 30000,              // 轮询间隔（默认 30 秒）
  completionThresholdMs: 120000   // 完成阈值（默认 2 分钟）
});
```

## 📝 注意事项

1. **幂等性保证**：使用内存缓存记录已处理的 Subagent，服务重启后缓存会清空
2. **误判防护**：2 分钟阈值可根据实际情况调整
3. **日志监控**：服务会输出详细的日志，便于排查问题
