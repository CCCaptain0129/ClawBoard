## 开发原则

### 1. 小步快跑
每完成一个小任务就进行测试，测试通过后部署。

### 2. 测试优先
开发新功能前先编写测试用例，功能完成后立即运行测试。

### 3. 定期推送
每完成一个功能模块就推送到 GitHub，确保远程有备份。

### 4. 文档同步
代码变更后立即更新相关文档（PRD、PROGRESS、TASKS）。

### 5. 任务看板同步
每个任务状态变更后同步到 TASKS.md，使用 API 触发同步。

## 重要教训

### Git checkout 导致数据丢失
- **日期**: 2026-03-10 20:48
- **事件**: 意外执行 `git checkout .` 和 `rm -rf src/`，导致源代码全部丢失
- **教训**:
  - 修改代码前先提交到 git
  - 推送到 GitHub 确保远程有备份
  - 重要代码立即备份
